Contact Service API

A Spring Boot REST API for managing contacts with support for JSON, XML, CSV, and plain text formats.

About the Application

This Contact Service API provides a comprehensive solution for managing contact information. Built with Spring Boot and using SQLite for data persistence, it offers:

  • CRUD Operations: Create, read, update, and delete contact records
  • Content Negotiation: Get responses in JSON, XML, CSV, or plain text formats
  • Pagination: Control the amount of data returned in listing requests
  • OpenAPI Documentation: Interactive documentation for exploring and testing the API
  • SQLite Database: Lightweight, file-based database for persistent storage
  • Containerization: Docker support for easy deployment

Available Endpoints

GET /api/contacts

Returns a paginated list of all contacts.

Query Parameters:

  • page: Page number (default: 1)
  • limit: Number of items per page (default: 10)
  • format: Response format (json, xml, csv, txt)
JSON XML CSV Text

GET /api/contacts/{id}

Returns a specific contact by its ID.

Path Parameters:

  • id: The ID of the contact to retrieve

Query Parameters:

  • format: Response format (json, xml, csv, txt)
JSON XML CSV Text

POST /api/contacts

Creates a new contact.

Request Body: Contact object in JSON or XML format

JSON XML

PUT /api/contacts/{id}

Updates an existing contact.

Path Parameters:

  • id: The ID of the contact to update

Request Body: Updated contact object in JSON or XML format

JSON XML

DELETE /api/contacts/{id}

Deletes a contact.

Path Parameters:

  • id: The ID of the contact to delete

Content Negotiation

This API supports multiple response formats:

  • Use the format query parameter: /api/contacts?format=xml
  • Or use the Accept header in your request

Supported formats:

  • JSON (default): /api/contacts?format=json
  • XML: /api/contacts?format=xml
  • CSV: /api/contacts?format=csv
  • Text: /api/contacts?format=txt
